# CLAUDE.md

Guidance for Claude Code when working in this repository.

## What this project is

An empirical study comparing **bug-fix pull requests** authored by AI coding agents (Copilot, Cursor, Claude Code, Devin) against human-authored fixes on GitHub. Output is a set of figures answering 8 research questions, organized into 3 themes / notebooks.

- **Dataset:** [`mabujadallah/GitHub-Agentic-PR-Dataset`](https://huggingface.co/datasets/mabujadallah/GitHub-Agentic-PR-Dataset) on HuggingFace. Read directly via parquet URLs. `fix_classified_prs.parquet` is **~1.5 GB**; if a local copy exists under `.cache/` (or `$CLI_DATA_CACHE`), `analysis_utils._source` uses it automatically to avoid re-downloading.
- **Key columns:** repo is `repo_name` (exposed as `repo`); PRs have no `auto_merge`/`merged_by`/`review_comments_count`; the commits table has **no commit timestamp**.
- **Scope:** closed bug-fix PRs (`type == 'fix'`, `state == 'closed'`), excluding OpenAI Codex.
- **Coverage:** Dec 2024 – Feb 2026 (15 months), single uniform collection pipeline.
- **AIDev cutoff:** 2025-07-31 — splits the timeline into AIDev vs post-AIDev periods (RQ8). RQ8 also restricts to the **AIDev repo set** so the period comparison is on a fixed denominator.

## Layout

- [analysis_utils.py](analysis_utils.py) — shared module. Owns data loading, the AIDev cutoff, agent list and colour palette, stats helpers (chi-square, Mann-Whitney, significance labels), and plot style / save helpers. All notebooks import from here.
- [theme1_adoption_trends.ipynb](theme1_adoption_trends.ipynb) — RQ1 (monthly volume) and RQ7 (market share).
- [theme2_quality_over_time.ipynb](theme2_quality_over_time.ipynb) — RQ2 (merge rate), RQ3 (time-to-merge), RQ4 (patch size), RQ5 (revision burden).
- [theme3_agent_comparison.ipynb](theme3_agent_comparison.ipynb) — RQ6 (best agent overall) and RQ8 (AIDev vs post-AIDev).
- [investigate.py](investigate.py) — ad-hoc diagnostic script for spot-checking anomalies (e.g. Jul-25 spike, Copilot revision rate, Devin sparsity). Not part of the figure pipeline.
- [results/](results/) — generated PNG figures, one folder per theme. Regenerated by re-running the notebooks.

## Conventions worth knowing

- **Constants in `analysis_utils.py` are the source of truth.** `AGENTS`, `AGENT_COLORS`, `AIDEV_END`, `MIN_N_PROP`/`MIN_N_MEDIAN`, `MODEL_RELEASES`, and the `THEME*_DIR` paths must not be redefined inside notebooks.
- **Minimum support per monthly cell:** `MIN_N_PROP` (30) for rates, `MIN_N_MEDIAN` (20) for medians — otherwise `None`. (Replaced the old n≥5 floor, which produced noisy 1–2-PR points.)
- **Confidence intervals are mandatory on trend/summary stats.** Monthly rates carry `wilson_ci`; monthly medians carry `bootstrap_median_ci`; plotted as shaded `fill_between` bands.
- **Statistics always come with an effect size and FDR correction.** Proportions: `chi_square` + `odds_ratio_ci`. Continuous: `mann_whitney` + `cliffs_delta`. Collect raw p-values per family and pass through `bh_correct` before `sig_label`. At this N, raw p-values are meaningless on their own.
- **Repo-matched baseline:** agent-vs-human comparisons report both the unmatched and `build_matched_human_baseline` numbers; a divergence flags repo-mix confounding.
- **Patch size is split by file role** via `classify_file_role` (prod / test / generated) — never report a single lines-added number.
- **Survivorship:** `load_fix_prs` drops the last `SURVIVORSHIP_CUTOFF_DAYS` (30) by default.
- **Figures:** PNG only, 150 DPI, saved via `save_fig(fig, name, folder)`. Filename convention: `rq{N}_{slug}.png`. Red dotted `2025-07` AIDev line on every monthly chart; grey `MODEL_RELEASES` lines via `annotate_model_releases`.
- **Revision metric:** "revision effort" excludes the first commit per PR (`build_revision_stats`). NOTE: the commits table has no timestamp, so "first commit" relies on parquet row order — a known fragility.

## Known data limitations (do not silently work around)

- **No auto-merge signal** (`auto_merge`/`merged_by`/`review_comments_count` all absent, no commit timestamp): merge-rate and time-to-merge partly reflect repo merge policy. Stated as a threat-to-validity, not corrected.
- **No review-comment counts:** revision burden uses commit count + revision lines only.
- **`type=='fix'` label is unvalidated:** theme3 exports `results/label_validation_sample.csv` for manual labelling + a precision sensitivity check.

## Running the analysis

```powershell
# from repo root, with .venv activated
pip install matplotlib seaborn scipy pyarrow fsspec requests pandas
jupyter nbconvert --to notebook --execute theme1_adoption_trends.ipynb --inplace
jupyter nbconvert --to notebook --execute theme2_quality_over_time.ipynb --inplace
jupyter nbconvert --to notebook --execute theme3_agent_comparison.ipynb --inplace
```

Figures land in `results/theme{1,2,3}_figures/`.

## When adding a new RQ

1. Add the loading / aggregation logic to the relevant theme notebook (do not create a new notebook unless the theme is genuinely new).
2. If the helper is reusable, put it in [analysis_utils.py](analysis_utils.py) — never duplicate filtering/loading code in notebooks.
3. Use `AGENT_COLORS` for colour and `set_plot_style()` for style. Save via `save_fig`.
4. Apply the `MIN_N_PROP`/`MIN_N_MEDIAN` minimum and the AIDev cutoff annotation if monthly.
5. Pair every test with an effect size (`odds_ratio_ci`/`cliffs_delta`), `bh_correct` the family of p-values, and show CIs (`wilson_ci`/`bootstrap_median_ci`).
